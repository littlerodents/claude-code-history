### 交付物 2：爆款文章生成框架

**项目位置**: `/workspace/viral-article-generator/`

**框架组成**:

#### 1. 模板系统
- `templates/tech-tool-template.md` - 技术工具类模板
- 支持 Handlebars 变量语法
- 可扩展其他模板（效率方法论、实操教程等）

#### 2. 配置系统
- `examples/clawdbot-config.json` - Clawdbot 文章配置示例
- JSON 格式，支持完整的文章结构定义
- 包含所有必要字段（meta、hook、project、use_cases、installation、cta 等）

#### 3. 脚本工具
- **generate.js** - 文章生成脚本
  - 读取配置文件
  - 渲染 Handlebars 模板
  - 生成最终文章
  - 支持自定义输出路径

- **validate.js** - 配置验证脚本
  - JSON 格式验证
  - 必填字段检查
  - 字段类型验证
  - 统计信息输出

#### 4. 项目配置
- `package.json` - Node.js 项目配置
  - 依赖: handlebars ^4.7.8
  - npm scripts: generate, validate, test

- `README.md` - 详细使用文档
  - 快速开始指南
  - 配置文件结构说明
  - 命令行工具使用
  - 最佳实践和写作技巧

#### 5. 输出目录
- `output/` - 生成的文章存放目录
- 示例: `output/clawdbot-config.md`

## 🚀 如何使用框架

### 快速开始

1. **安装依赖**
```bash
cd /workspace/viral-article-generator
npm install
```

2. **验证配置**
```bash
npm run validate examples/clawdbot-config.json
```

3. **生成文章**
```bash
npm run generate examples/clawdbot-config.json
```

4. **查看生成的文章**
```bash
cat output/clawdbot-config.md
```

### 创建自己的文章

1. **复制示例配置**
```bash
cp examples/clawdbot-config.json examples/my-project-config.json
```

2. **编辑配置文件**
根据你的项目修改以下关键字段：
- `meta.title` - 文章标题
- `hook.*` - 开头钩子内容
- `project.*` - 项目信息
- `use_cases` - 使用案例列表
- `installation.steps` - 安装步骤
- `cta.benefits` - 福利和行动引导

3. **验证配置**
```bash
npm run validate examples/my-project-config.json
```

4. **生成文章**
```bash
npm run generate examples/my-project-config.json
```

## 📖 爆款文章结构解析

### 标题公式
**「新奇概念 + 实用价值承诺」**

示例:
- ✅ 像写代码一样操控电脑：我发现了一个开源"贾维斯"（附保姆级安装教程）
- ✅ 告别只会聊天的 AI：这个开源项目让我在床上用手机遥控电脑干活
- ❌ Clawdbot 使用教程（太平淡）

### 7 步框架

1. **钩子（200 字）**
   - 日常场景代入
   - 制造悬念/好奇心
   - 视觉冲击

2. **冲击（300 字）**
   - 展示效果
   - 制造反差
   - 社交证明

3. **揭秘（400 字）**
   - 背后的工具/方法
   - 核心能力介绍
   - 技术架构

4. **实操（600 字）**
   - 亲身体验
   - 2-3 个真实场景
   - 真实感受

5. **强化（500 字）**
   - 更多用例
   - 证明可复用
   - 震撼功能

6. **教程（800 字）**
   - 保姆级步骤
   - 可复制代码
   - 常见问题

7. **福利（200 字）**
   - 免费资源
   - 明确的 CTA
   - 降低门槛

## 🎨 配置文件核心字段

### meta（元信息）
```json
{
  "template": "tech-tool-template",
  "title": "文章标题（20-50字）",
  "word_count": "3500-4000 字",
  "reading_time": "8-10 分钟"
}
```

### hook（钩子）
```json
{
  "scenario": "日常场景描述",
  "discovery": "发现什么神奇的东西",
  "surprise": "惊喜元素"
}
```

### project（项目信息）
```json
{
  "name": "项目名称",
  "tagline": "一句话介绍",
  "github_url": "https://github.com/...",
  "website": "https://example.com",
  "social_proof": ["推荐人", "用户数等"]
}
```

### use_cases（使用案例）
```json
[
  {
    "title": "案例标题",
    "command": "用户命令",
    "result": "执行结果",
    "impact": "带来的价值"
  }
]
```

### installation（安装教程）
```json
{
  "prerequisites": ["前置要求1", "前置要求2"],
  "steps": [
    {
      "step": 1,
      "title": "步骤标题",
      "command": "执行命令",
      "validation": "验证方式",
      "screenshot": true
    }
  ],
  "common_issues": [
    {
      "question": "常见问题",
      "answer": "解决方案"
    }
  ]
}
```

### cta（行动号召）
```json
{
  "benefits": [
    {
      "title": "福利标题",
      "description": "福利描述",
      "action": "行动引导"
    }
  ]
}
```

## 💡 使用建议

### 适用场景
✅ 技术工具介绍
✅ 开源项目推广
✅ 产品功能演示
✅ 实操教程撰写

### 不适用场景
❌ 纯理论文章
❌ 新闻报道
❌ 学术论文
❌ 短篇内容（<2000 字）

### 最佳实践
1. **先写配置，再生成文章**
   - 配置文件强制你思考文章结构
   - 确保不漏掉关键要素

2. **人工润色必不可少**
   - 生成的文章是骨架
   - 需要添加个人化表达
   - 补充细节和情感

3. **A/B 测试标题**
   - 在配置中准备 2-3 个标题
   - 测试不同的表达方式
   - 选择点击率最高的

4. **素材准备要充分**
   - 截图和 GIF 是灵魂
   - 提前规划拍摄内容
   - 确保视觉冲击力
